# Editor i18n + RTL Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize editor chrome via a module i18n singleton (`t()` + built-in `en` catalog + host `messages`), add RTL `dir` support, and replace all hardcoded chrome strings with `t('key')` lookups.

**Architecture:** A singleton i18n module configured by `<pigeon-editor>` before first render; `t(key)` resolves active-locale → en → key. `dir` is set on the host (derived from locale or `config.dir`), flipping the shell layout. Comprehensive string extraction is done area-by-area; because each `en` value equals the current hardcoded string, existing tests stay green.

**Tech Stack:** TypeScript, Lit 3, Vitest.

**Spec:** `docs/superpowers/specs/2026-06-17-i18n-rtl-design.md`
**Branch:** `feat/21-i18n-rtl` (stacked on `feat/24-special-links`).

## Extraction recipe (used by Tasks 3–7)

For each component in the batch:
1. Add `import { t } from '<relative>/i18n/index.js';` (compute the relative path: e.g. from `components/properties/panels/X.ts` it's `../../../i18n/index.js`; from `components/toolbar/X.ts` it's `../../i18n/index.js`; from `rich-text/ui/X.ts` it's `../../i18n/index.js`).
2. For every USER-FACING chrome string, add a key to `packages/editor/src/i18n/en.ts` whose **value is the exact current string**, and replace the literal with `t('key')`:
   - Text node `>Label<` → `>${t('key')}<`
   - `title="Label"` → `title=${t('key')}`, `label="Label"` → `.label=${t('key')}` (or `label=${t('key')}`), `placeholder="..."` → `placeholder=${t('key')}`, `aria-label="..."` → `aria-label=${t('key')}`
   - Heading/option text likewise.
3. **Do NOT extract:** non-UI strings — event names, CSS, attribute *values* that aren't shown to users (e.g. `type="url"`, `drag-type="..."`), merge-tag tokens, block `type` ids, or dynamic data (block content, user input).
4. **Key convention:** dotted, grouped by area: `toolbar.*`, `palette.*`, `preview.*`, `template.*`, `asset.*`, `panel.<block>.*` (e.g. `panel.button.linkUrl`), `control.<name>.*`, `richtext.*`. Reuse a shared key when the same label appears in multiple places (e.g. `common.delete`).
5. Keep `en.ts` keys sorted/grouped by area with a comment header per area.

**Verification per extraction task:** the component(s)' EXISTING tests must still pass unchanged (en values are identical to the old literals). Run the touched components' test files + `npm test` for the package.

```ts
 * Built-in English catalog — the source of truth for every editor chrome

 * string the component previously hardcoded.

  'toolbar.redo': 'Redo',
  'toolbar.preview': 'Preview',
};
```

Create `packages/editor/src/i18n/index.ts`:

```ts
import { EN_MESSAGES } from './en.js';

type Catalog = Record<string, string>;

/** Language subtags that render right-to-left. */
const RTL_LANGS = new Set(['ar', 'he', 'fa', 'ur', 'ps', 'sd', 'dv', 'yi']);

let _locale = 'en';
let _catalogs: Record<string, Catalog> = { en: EN_MESSAGES };

/** Configure the active locale and merge host catalogs over the en baseline. */
export function configureI18n(locale?: string, messages?: Record<string, Catalog>): void {
  _locale = locale || 'en';
  _catalogs = { en: EN_MESSAGES, ...(messages ?? {}) };
}

/** Translate a key: active locale → en fallback → the key itself. */
export function t(key: string): string {
  return _catalogs[_locale]?.[key] ?? EN_MESSAGES[key] ?? key;
}

export function getLocale(): string {
  return _locale;
}

/** Resolve text direction: explicit override wins, else derived from the locale's language subtag. */
export function resolveDir(locale?: string, override?: 'ltr' | 'rtl'): 'ltr' | 'rtl' {
  if (override) return override;
  const lang = (locale || 'en').toLowerCase().split(/[-_]/)[0];
  return RTL_LANGS.has(lang) ? 'rtl' : 'ltr';
}
```

In `packages/core/src/types/editor.ts`, add to `EditorConfig` (after `linkTypes`):
```ts
  /** Active locale code (e.g. 'fr', 'ar'). Defaults to 'en'. */
  locale?: string;

  messages?: Record<string, Record<string, string>>;
  /** Explicit text direction. When unset, derived from `locale`. */
```
